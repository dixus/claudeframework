# Spec: AI Maturity Platform Enhancements

## Goal

Extend the results page with contextual benchmarks, per-level how-to playbooks, and a PDF export button; add a standalone `/glossary` page — so users understand where they stand against research data, know the concrete next step, and can download or share their report.

---

## Background: framework alignment

The research papers (AMF v4.5.1, Scaling Stack v2.0, Playbook v1.0) are the authoritative source for all benchmark values and playbook content in this spec. Key facts extracted:

| Level | θ range (app scale) | Label | ARR/Employee | Time to €100M |
|---|---|---|---|---|
| 0 | 0–20 | Traditional | €150–200K | 60+ months |
| 1 | 21–50 | AI-Powered | €200–400K | 36–60 months |
| 2 | 51–80 | AI-Enabled | €400–700K | 24–36 months |
| 3 | 81–100 | AI-Native | €700–1.5M | 18–24 months |

Research dataset: n=62 companies (2021–2025), B2B SaaS, post-Series A.
Distribution: Level 1: 37%, Level 2: 29%, Level 3: 15%, Level 0 baseline: 24%.

Three intervention models (from Scaling Stack v2.0):
- **Model 1 — Bottleneck Resolution**: 8–12 weeks; fix the single blocking dimension
- **Model 2 — Stage Transition**: 90 days; advance toward next level boundary
- **Model 3 — Level Transition**: 6–24 months; full architectural transformation

Note: Research papers use Architecture weight 20%; the app (per spec.txt) uses Architecture 15% / Strategy 20%. Keep the app weights — this is a known intentional divergence.

## Requirements

### Feature A — Insights Panel (contextual, results page)

- Appears on the results page below the existing ScoreCard
- Shows a "Benchmark" section with:
  - User's level label + θ score vs. level mean from research dataset (Level 0 mean: θ=10, L1: θ=35, L2: θ=65, L3: θ=90)
  - ARR/Employee range for user's level (from table above)
  - Time to €100M estimate for user's level
  - "Next level target": minimum θ needed to reach next level, and the gap
- Shows a "Peer distribution" bar: % of 62-company dataset at each level (as simple percentage labels, not a chart)
- All values are static (no API) — sourced from `src/lib/scoring/benchmarks.ts`

### Feature B — Playbook Panel (contextual, results page)

- Appears on results page below the BottleneckPanel
- Recommends the most relevant intervention model based on the user's result:
  - If gap to next level < 15 points AND bottleneck dimension < 60 → **Model 1** (Bottleneck Resolution)
  - If gap to next level 15–30 points → **Model 2** (Stage Transition)
  - If gap to next level > 30 points OR user is Level 3 → **Model 3** (Level Transition)
- For the recommended model, show:
  - Model name + timeline (e.g. "Bottleneck Resolution — 8–12 weeks")
  - 1-sentence description of the approach
  - 4–5 concrete first steps, specific to the user's bottleneck dimension
  - Expected outcome (quantified where possible, from research papers)
- All content is static, sourced from `src/lib/scoring/playbooks.ts`
- Data structure: `Record<DimensionKey, Record<ModelId, PlaybookContent>>`

### Feature C — PDF Export

- "Download Report" button in the results page header (next to company name)
- Client-side PDF generation using `jsPDF` (no server)
- PDF content (single page, A4):
  - Header: company name, θ score, level label, date
  - Radar chart captured as image via SVG serialization (`XMLSerializer` → canvas → PNG); html2canvas was removed because it cannot parse `oklch()` colors generated by Tailwind v4
  - Dimension scores table (6 rows: dimension / score / gap to 70)
  - Bottleneck section: dimension name + 3 recommended actions
  - Footer: "AI Maturity Platform — ai-maturity.app"
- Filename: `ai-maturity-<company-name-slugified>.pdf`
- Button shows a loading spinner during generation (async)

### Feature D — Glossary page

- Route: `/glossary`
- Standalone page (not inside the assessment flow)
- Navigation: add a "Glossary" link to the app header/footer
- 18 terms defined, sourced from `src/lib/scoring/glossary.ts`:
  - θ_index (theta index)
  - AI Maturity Level
  - Level 0 — Traditional
  - Level 1 — AI-Powered
  - Level 2 — AI-Enabled
  - Level 3 — AI-Native
  - Superlinear Scaling
  - Bottleneck Dimension
  - Gating Rule
  - ARR/Employee (θ_observed)
  - Dimension Score
  - Weighted Score
  - Substitution (Level 1 mechanism)
  - Augmentation (Level 2 mechanism)
  - Orchestration (Level 3 mechanism)
  - Transition Readiness
  - Bottleneck Resolution (Model 1)
  - Stage Transition (Model 2)
- Each term: `{ term: string, definition: string, example?: string }`
- Page layout: alphabetically sorted, anchor-linked, searchable via browser Cmd+F

## Out of scope

- Dynamic benchmarks from an external API or database
- User accounts, saved assessments, or multi-session history
- Animated transitions or interactive playbook steps
- CRM/email capture (Phase 1 roadmap item)
- Reviewer mode or evidence upload (Phase 2)

## Implementation notes

- `jsPDF` is client-side only — `PdfExportButton` uses `'use client'` and dynamic import inside the click handler
- The radar chart DOM element needs a stable `id` (`id="radar-chart"`) so `XMLSerializer` can target it via `document.getElementById`; added to `RadarChartPanel.tsx`
- **html2canvas was removed**: it cannot parse `oklch()` colors from Tailwind v4. The radar chart is captured by serializing the Recharts SVG via `XMLSerializer`, drawing it to an `<canvas>`, and exporting as PNG
- Playbook model selection logic belongs in `src/lib/scoring/playbooks.ts` as a pure function `selectModel(result: AssessmentResult): ModelId` — not in the component
- `benchmarks.ts` must remain framework-agnostic (no React) — same constraint as `src/lib/scoring/`
- All benchmark values in this spec come from AMF v4.5.1 (n=62 dataset) — cite the source in code comments

- Clicking "Download Report" produces a downloadable PDF within 5 seconds; the PDF contains the company name, θ score, and dimension table
- Navigating to `/glossary` renders 18 terms in alphabetical order, each with a definition
- A "Glossary" link in the header/footer navigates to `/glossary`
- No TypeScript errors (`npx tsc --noEmit` passes)
- No ESLint errors (`npm run lint` passes)
