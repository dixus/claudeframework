## Not proposed (considered and rejected)

- **`effort:` frontmatter**: Changelog claims skill support (v2.1.80) but official docs don't list it. Not proposing until verified.
- **`context: fork`**: Some skills mention subagent execution in their body text but are designed to be orchestrated by the user or `/ship`, not auto-forked. Adding `context: fork` would change invocation semantics.
- **`license` / `metadata` fields**: Only relevant if distributing as a plugin. Current framework is copied as a directory, not installed via `/plugin add`.
- **`disallowedTools` on subagents**: Redundant with `allowed-tools` on the skill itself. Prefer the simpler approach.
