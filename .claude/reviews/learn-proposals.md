### Skill: 2_review

**What:** Add `disallowedTools` frontmatter to enforce read-only review (belt-and-suspenders)
**Why:** v2.1.78 introduced `disallowedTools` frontmatter. The skill already uses `allowed-tools` whitelist, but `disallowedTools` is the canonical blocklist. Using both provides defense in depth — if a new tool is added to Claude Code, `allowed-tools` whitelists it out, but `disallowedTools` also explicitly blocks the dangerous ones.
**Diff:**

```
old (line 8):
allowed-tools: Read, Glob, Grep, WebFetch, WebSearch, Agent

new (add line after allowed-tools):
allowed-tools: Read, Glob, Grep, WebFetch, WebSearch, Agent
disallowedTools: Edit, Write, Bash, NotebookEdit, MultiEdit
```

### Skill: 3_fix

**What:** Add `maxTurns: 15` frontmatter as a hard circuit breaker
**Why:** v2.1.78 introduced `maxTurns` for skills/agents. The skill has a soft circuit breaker (recurring issue detection in step 5), but no hard turn limit. If the soft breaker fails (e.g., each cycle produces a different error), the skill can spin indefinitely. `maxTurns: 15` provides a guaranteed stop.
**Diff:**

```

---

new:

...
effort: high
maxTurns: 15
---
```
