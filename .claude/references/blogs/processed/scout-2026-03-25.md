## Official Anthropic skills repo analysis

| Field/Convention                         | Official spec                                             | Our framework                                              | Gap                                                                              |
| ---------------------------------------- | --------------------------------------------------------- | ---------------------------------------------------------- | -------------------------------------------------------------------------------- |
| `name` (required)                        | lowercase-with-hyphens, max 64 chars, must match dir name | Used but not enforced to match dir name                    | Minor — some skills use underscores in dir names (`0_spec`, `1_implement`, etc.) |
| `description` (required)                 | Max 1024 chars, third-person, includes when-to-use        | Used consistently                                          | No gap                                                                           |
| `license` (optional)                     | License name or reference                                 | Not used                                                   | No gap — internal framework                                                      |
| `compatibility` (optional)               | Max 500 chars, environment requirements                   | Not used                                                   | No gap — single-project framework                                                |
| `metadata` (optional)                    | Arbitrary key-value pairs                                 | Not used                                                   | No gap — not needed                                                              |
| `allowed-tools` (optional, experimental) | Space-delimited tool list                                 | Not used                                                   | Low priority — we use `tools` in agent definitions                               |
| `scripts/` directory                     | Executable code                                           | Not used (skills are instruction-only)                     | No gap — our skills don't need scripts                                           |
| `references/` directory                  | Additional documentation                                  | Not used in skills (we use `.claude/references/` globally) | No gap — different architecture                                                  |
| `assets/` directory                      | Static resources                                          | Not used                                                   | No gap — not needed                                                              |
| `disable-model-invocation` (Claude Code) | Prevents auto-invocation                                  | Used in several skills                                     | No gap                                                                           |
