### Proposal 1 ‚Äî Update /create-hook with v2.1.89 hook features

**File:** `.claude/skills/create-hook/SKILL.md`
**What:** Add `PermissionDenied` event, `prompt`/`agent` hook types, and `defer` permission decision to the reference
**Why:** v2.1.89 added `PermissionDenied` hook event and `defer` permission decision. v2.1.63 added `prompt` and `agent` hook handler types that aren't documented in our skill. Users creating hooks need to know the full set of options.
**Diff:**
```
old: Hook event list ends at existing events; hook types only mention command/http

new: Add to event reference:
- PermissionDenied (v2.1.89) ‚Äî fires after auto mode classifier denies a tool call
- WorktreeCreate / WorktreeRemove ‚Äî fires on worktree lifecycle events
- ConfigChange ‚Äî fires when configuration files change during a session
- TeammateIdle ‚Äî fires when an agent team teammate is about to go idle

Add to hook handler types:
- prompt: Send prompt to Claude for single-turn evaluation (LLM-as-judge)
- agent: Spawn a subagent to verify conditions before returning a decision

Add to PreToolUse permissionDecision values:
- "defer": Pause at tool call, resume later with `-p --resume`
```

### Proposal 4 ‚Äî Add MCP_CONNECTION_NONBLOCKING to /ship CI guidance

**File:** `.claude/skills/ship/SKILL.md`
**What:** Add `MCP_CONNECTION_NONBLOCKING=true` env var to `--bare` CI guidance
**Why:** v2.1.89 added this env var to skip MCP connection wait in `-p` mode. Combined with `--bare`, this makes CI runs faster by not waiting for MCP servers that aren't needed in headless mode.
**Diff:**
```
old: E) Run headless in CI ‚Äî use `claude --bare -p "/ship <feature>"`

new: E) Run headless in CI ‚Äî use `MCP_CONNECTION_NONBLOCKING=true claude --bare -p "/ship <feature>"` (skips hooks/LSP/MCP wait)
```
